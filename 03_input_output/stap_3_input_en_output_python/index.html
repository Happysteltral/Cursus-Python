    Stap 3 ‚Äì Input en Output (Python)

    3.1 Waarom input en output?

    Tot nu toe hebben we gewerkt met vaste waarden in ons programma. Dat is handig om te oefenen, maar echte programma‚Äôs worden pas interessant wanneer: - üë§ de gebruiker iets kan invoeren (input) - üñ•Ô∏è het programma iets terug toont (output)

    üëâ In deze stap leren we hoe een Python-programma kan communiceren met de gebruiker.

    3.2 Output tonen met print()

    3.2.1 Basisgebruik

    Met print() toon je tekst of waarden op het scherm.

    print("Hallo wereld")

    Je kan ook een variabele afdrukken:

    leeftijd = 16
print(leeftijd)

    3.2.2 Tekst en variabelen combineren

    ‚ùå Dit werkt niet:

    print("Ik ben " + leeftijd + " jaar oud")

    Waarom niet? üëâ leeftijd is een getal, geen tekst.

    ‚úÖ Oplossing 1: komma‚Äôs gebruiken (simpel en veilig)

    print("Ik ben", leeftijd, "jaar oud")

    ‚úÖ Oplossing 2: str() gebruiken

    print("Ik ben " + str(leeftijd) + " jaar oud")

    3.2.3 f-strings (aanbevolen)

    Dit is de modernste en duidelijkste manier:

    print(f"Ik ben {leeftijd} jaar oud")

    üëâ Alles tussen { } wordt automatisch omgezet naar tekst.

    3.3 Input vragen met input()

    3.3.1 Basisgebruik

    naam = input("Wat is je naam? ")
print(f"Hallo {naam}")

    ‚ö†Ô∏è Belangrijk: alles wat met input() binnenkomt is tekst (string).

    3.3.2 Rekenen met input

    ‚ùå Dit gaat fout:

    leeftijd = input("Wat is je leeftijd? ")
volgend_jaar = leeftijd + 1

    Waarom? üëâ leeftijd is een string.

    ‚úÖ Correcte aanpak:

    leeftijd = int(input("Wat is je leeftijd? "))
volgend_jaar = leeftijd + 1
print(f"Volgend jaar ben je {volgend_jaar} jaar")

    3.4 Veelgemaakte fouten

    Fout Verklaring
    input() vergeten om te zetten Input is altijd tekst
    + gebruiken bij tekst en getallen Typefout
    Geen duidelijke vraag stellen Slechte gebruikerservaring

    Oefening 2 ‚Äì Leeftijd

    3.6 Samenvatting

    • print() = output tonen
    • input() = gegevens vragen aan de gebruiker
    • Input is altijd tekst
    • Gebruik int() of float() voor berekeningen
    • f-strings maken code leesbaar
